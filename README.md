### 1. Encapsulation ðŸ›¡ï¸
The process of bundling data (attributes) and methods (functions) into a single unit called a **Class**, while restricting direct access to some components.
* **Goal:** Data protection and integrity (Information Hiding).
* **Key Concepts:** `Access Modifiers`, `Constructors`, `Destructors`, and `this pointer`.



---

### 2. Abstraction â˜ï¸
Focusing on "what" an object does rather than "how" it does it. It hides implementation details and shows only the necessary interface.
* **Goal:** Reducing complexity and separating design from implementation.
* **Key Concepts:** `Abstract Classes` and `Pure Virtual Functions`.



---

### 3. Inheritance ðŸ§¬
A mechanism where a new class (Child) acquires properties and behaviors from an existing class (Parent).
* **Goal:** Code Reusability and hierarchical classification.
* **Key Concepts:** `Base & Derived Classes`, `Protected Access`, and `Inheritance Modes`.



---

### 4. Polymorphism ðŸŽ­
The ability of an object or function to take on multiple forms, allowing a single interface to represent different underlying types.
* **Goal:** Flexibility and extensibility.
* **Key Concepts:** `Virtual Functions` (Runtime) and `Function/Operator Overloading` (Compile-time).
